## ScraperAPI 到底是什么

简单说，它是一个 Web Scraping API 服务。你把目标 URL 丢给它的 API 端点，它帮你处理代轮换、浏览器指纹、验证码绕过、请求头伪装这些脏活累活，返回给你干净的 HTML 或 JSON 数据。

它不是一个可视化爬虫工具，没有拖拽界面。你需要写代码调用它的 API。Python、Node.js、Java、Ruby 都行，本质上就是一个 HTTP 请求。

对于亚马逊评论抓取这个场景，ScraperAPI 有专门的 Amazon 结构化数据端点。你传入 ASIN，它直接返回解析好的 JSON 格式评论数据——评分、标题、正文、日期、是否验证购买，都帮你提取好了。不用自己写 XPath 或 CSS 选择器去解析 HTML。

## 常见问题

### ScraperAPI 抓亚马逊评论会被封号吗？

ScraperAPI 用的是它自己的代理池，请求不经过你的 IP。你的亚马逊账号（如果有的话）和抓取行为之间没有关联。被封的是代理 IP，ScraperAPI 会自动轮换，这是它的核心能力。

5000 次请求，如果你用结构化数据端点抓评论，大概能抓 500-1000 条评论（取决于每个产品的评论页数）。跑通一个完整的 POC 够了。

### 支持抓取亚马逊哪些站点？

美国、英国、德国、法国、日本、印度、加拿大、澳大利亚等主要站点都支持。通过 `country_code` 参数切换。

### 返回的数据格式是什么？

结构化端点返回 JSON，包含评分、标题、评论正文、日期、作者名、是否验证购买、评论所属的产品变体等字段。普通模式返回原始 HTML。

### 请求失败了怎么办？

返回非 200 状态码的请求不计入你的月度额度。你可以在代码里加重试逻辑，通常重试一次就能成功。
